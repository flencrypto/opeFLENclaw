REQUIRED KEYS AND LOGINS — OpenClaw
Generated: 2026-03-06
Repository: https://github.com/openclaw/openclaw

This file documents every API key, token, OAuth credential, and external login
that OpenClaw can use, grouped by category. Only the items you actually plan to
use need to be configured. Everything is optional except "at least one AI model
provider" for the gateway to function.

Copy env vars into:
  - .env (local dev, repo-relative)
  - ~/.openclaw/.env (daemon / launchd / systemd installs)
  - OR directly into ~/.openclaw/openclaw.json under the relevant config key

See .env.example in the repo root for a copy-paste template.

===============================================================================
SECTION 1 — GATEWAY AUTH (Recommended for any non-loopback deployment)
===============================================================================

If the gateway binds to 0.0.0.0 or any interface reachable over a network,
set at least one of the following. Without auth, any local process can connect.

  OPENCLAW_GATEWAY_TOKEN=<random-hex-string>
    Purpose : Bearer-token auth for the gateway WebSocket/HTTP API.
    Generate: openssl rand -hex 32
    Used by : All clients connecting to the gateway (web UI, mobile apps, CLI).

  OPENCLAW_GATEWAY_PASSWORD=<strong-password>
    Purpose : Alternative to OPENCLAW_GATEWAY_TOKEN (use one or the other).
    Note    : Token auth is preferred; password auth is provided for convenience.

Optional path overrides:
  OPENCLAW_STATE_DIR=~/.openclaw
  OPENCLAW_CONFIG_PATH=~/.openclaw/openclaw.json
  OPENCLAW_HOME=~

SECTION 6 — CI/CD (GitHub Actions — release workflow)
===============================================================================

These secrets are only needed for the release workflow
(.github/workflows/release.yml) to build production artifacts.
Add them in: GitHub repo → Settings → Secrets and variables → Actions

--- Android release signing ---
  ANDROID_KEYSTORE_BASE64
    Content : Base64-encoded JKS/PKCS12 keystore file
    Generate:
      keytool -genkey -v -keystore release.jks -alias openclaw \
        -keyalg RSA -keysize 4096 -validity 10000
      base64 -i release.jks | tr -d '\n'  # Linux/macOS
      [Convert]::ToBase64String([IO.File]::ReadAllBytes("release.jks"))  # PowerShell

  ANDROID_STORE_PASSWORD
    Content : Password used when creating the keystore (-storepass)

  ANDROID_KEY_ALIAS
    Content : Key alias (e.g. "openclaw") used with -alias

  ANDROID_KEY_PASSWORD
    Content : Key password used when creating the key (-keypass)

Note: Without these four secrets, assembleRelease will still run but the APK
will not be signed and cannot be published to Google Play.

NOTES ON SERVER-ONLY HANDLING
===============================================================================

- All API keys and tokens are loaded server-side only (Node.js process env).
- The web control UI never receives raw credential values from the server.
- The `openclaw doctor` command diagnoses common misconfiguration issues.
- The `openclaw channels status --probe` command verifies channel connectivity.
- The `openclaw onboard` command provides an interactive wizard for first-time
  setup of AI providers and optional channel integrations.

For full documentation see: https://docs.openclaw.ai

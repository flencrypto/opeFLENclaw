REQUIRED KEYS AND LOGINS — OpenClaw
Generated: 2026-03-06
Source of truth: src/integrations/requirements.ts
All keys/tokens are SERVER-ONLY — never expose them client-side.

================================================================================
HOW TO CONFIGURE
================================================================================

1. Copy .env.example to ~/.openclaw/.env (or ./.env for dev runs).
2. Fill only the sections you need.
3. Restart the gateway after changes: openclaw gateway restart
4. Verify with: openclaw doctor
   or:          openclaw channels status --probe

Env var load order (highest → lowest priority):
  process env → ./.env → ~/.openclaw/.env → openclaw.json `env` block

--------------------------------------------------------------------------------
SECTION 1 — GATEWAY AUTH
--------------------------------------------------------------------------------

Feature : Gateway Authentication
Unlocks : Secures the HTTP gateway when exposed beyond localhost (required for
          remote access, cloud deploys, or Docker).

Required env vars:
  OPENCLAW_GATEWAY_TOKEN   Long random token clients must present to authenticate.
                           Generate: openssl rand -hex 32
                           (Optional when gateway is loopback-only.)

Where to get it:
  Generate locally — no external service needed.

Dependent routes / commands:
  openclaw gateway run
  openclaw channels status --probe

Server-only: token never forwarded to channel clients.

SECTION 2 — MODEL PROVIDERS  (set at least ONE)

OpenClaw routes agent requests to the first configured provider by default.
You only need to configure the providers you want to use.

──────────────────────────────────────────
2a. OpenAI
──────────────────────────────────────────
Unlocks : GPT-4o, o3, o4-mini, TTS with OpenAI voices.

Required env vars:
  OPENAI_API_KEY    Secret key starting with "sk-".

Where to get it:
  1. Sign in at https://platform.openai.com/
  2. Go to API Keys → Create new secret key.
  3. Copy the key and set OPENAI_API_KEY=sk-... in ~/.openclaw/.env

Official link: https://platform.openai.com/api-keys

Dependent: openclaw agent --model openai/...
           TTS (provider: openai)

Server-only: key is never forwarded.

Configure ONLY the channels you want to enable.

──────────────────────────────────────────
3a. Telegram
──────────────────────────────────────────
Required:
  TELEGRAM_BOT_TOKEN   Bot token from @BotFather (format: 123456:ABCDEF…).

Steps:
  1. Message @BotFather on Telegram → /newbot → follow prompts.
  2. Copy the token shown.
  3. Set TELEGRAM_BOT_TOKEN=<token> in ~/.openclaw/.env
     OR run: openclaw channels add telegram

Official: https://core.telegram.org/bots#how-do-i-create-a-bot
Docs: https://docs.openclaw.ai/channels/telegram

Commands: openclaw channels add telegram
          openclaw channels status --probe

──────────────────────────────────────────
3b. Discord
──────────────────────────────────────────
Required:
  DISCORD_BOT_TOKEN   Bot token from Discord Developer Portal.

Steps:
  1. https://discord.com/developers/applications → New Application.
  2. Bot → Add Bot → Reset Token → copy token.
  3. Enable Message Content Intent (Privileged Gateway Intents).
  4. OAuth2 → URL Generator → invite bot to server.
  5. Set DISCORD_BOT_TOKEN=... in ~/.openclaw/.env
     OR run: openclaw channels add discord

Official: https://discord.com/developers/applications
Docs: https://docs.openclaw.ai/channels/discord

3c. Slack
──────────────────────────────────────────
Required:
  SLACK_BOT_TOKEN    Bot OAuth token (xoxb-…).
  SLACK_APP_TOKEN    App-level token for Socket Mode (xapp-…).

Steps:
  1. https://api.slack.com/apps → Create New App → From scratch.
  2. OAuth & Permissions → add scopes: app_mentions:read, chat:write, etc.
  3. Install to workspace → copy Bot User OAuth Token (xoxb-…).
  4. App-Level Tokens → Generate with connections:write scope → copy (xapp-…).
  5. Set both tokens in ~/.openclaw/.env
     OR run: openclaw channels add slack

Official: https://api.slack.com/apps
Docs: https://docs.openclaw.ai/channels/slack

SECURITY NOTES
================================================================================

1. All keys/tokens are SERVER-ONLY. They are loaded by the gateway process and
   never forwarded to channel messages or exposed to users.

2. Store keys in ~/.openclaw/.env (not in the repo, not in openclaw.json unless
   you fully trust the config file location).

3. Rotate any key that may have been exposed; never commit real keys to git.

4. The gateway's security audit (openclaw doctor) reports missing or weak
   OPENCLAW_GATEWAY_TOKEN values.

5. For OAuth integrations, tokens are stored in auth-profiles under
   ~/.openclaw/ — protect that directory (chmod 700).
